### Introduction

The **Fast and Slow Pointers** pattern uses two pointers moving at different speeds in a linked structure.

Common setup:

* `slow` moves one step.
* `fast` moves two steps.

This speed gap reveals hidden structure such as cycles, middle points, and cycle entry nodes.

### Intuition

A brute-force cycle check often uses extra memory:

```java icon="code" title="Brute Force with Visited Set"
public boolean hasCycle(ListNode head) {
    Set<ListNode> visited = new HashSet<>();
    ListNode current = head;

    while (current != null) {
        if (visited.contains(current)) {
            return true;
        }
        visited.add(current);
        current = current.next;
    }

    return false;
}
```

Fast & Slow gets the same result in **O(1)** extra space.

### What are Predictable Dynamics?

If a cycle exists, `fast` gains one node per step relative to `slow` (2 vs 1). In a loop, this relative gain guarantees eventual meeting.

If no cycle exists, `fast` reaches `null` first.

For middle finding, every two steps of `fast` correspond to one step of `slow`, so when `fast` ends, `slow` is centered.

### Core Strategies

#### 1. Cycle Detection (Floyd’s Tortoise and Hare)

* Run two pointers at 1x and 2x speed.
* Meeting implies cycle.

#### 2. Middle Node Detection

* Same movement rule.
* When `fast` finishes, `slow` points to middle.

#### 3. Cycle Entry Detection

* After meeting, reset one pointer to head.
* Move both one step; next meeting is cycle start.

### Implementation Examples

#### Example A: Prefix-Based Rolling Computation

```java icon="code" title="Linear Data Structures - Prefix Running Sum"
public int[] runningSum(int[] nums) {
    int[] prefix = new int[nums.length];
    int running = 0;

    for (int i = 0; i < nums.length; i++) {
        running += nums[i];
        prefix[i] = running;
    }

    return prefix;
}
```

#### Example B: Sliding-Style Window Scan

```java icon="code" title="Linear Data Structures - Window Template"
public int maxSumWindow(int[] nums, int k) {
    int left = 0;
    int windowSum = 0;
    int best = Integer.MIN_VALUE;

    for (int right = 0; right < nums.length; right++) {
        windowSum += nums[right];

        if (right >= k - 1) {
            best = Math.max(best, windowSum);
            windowSum -= nums[left];
            left++;
        }
    }

    return best;
}
```


### Real-World Analogies

#### 1. Conceptual Analogy: Two Runners on a Circular Track

A faster runner eventually laps and meets a slower runner if the track loops. If the track is straight, they never meet again.

#### 2. Technical Analogy: Ring Buffer Pointer Monitoring

In circular buffering, producer and consumer pointers move at different rates. Their relative movement reveals wrap-around behavior and buffer state boundaries.

### When to Use (and How to Recognize It)

* Linked list traversal with constant-memory constraint.
* Need to detect loop/cycle without hash set.
* Need middle node in one pass.
* Need cycle entry or list splitting pivot.

### Pros & Cons

| Pros | Cons |
| --- | --- |
| Solves cycle and middle-node tasks in **O(n)** time, **O(1)** space. | Primarily effective on pointer-based linear structures. |
| No hash structures required. | Requires careful null checks (`fast` and `fast.next`). |
| Elegant one-pass behavior for multiple tasks. | Mathematical reasoning for cycle-entry may be non-obvious initially. |
