### Introduction

The **Two Pointers** pattern is an algorithmic technique that uses two indices (pointers) to traverse a linear data structure such as an array, string, or linked list.

A pointer is simply a variable that tracks a position. Most basic loops use one pointer, but adding a second pointer gives us more control:

1. **Simultaneous comparison:** Evaluate two positions at once.
2. **State separation:** Maintain a read position and a write position.
3. **Dynamic boundary control:** Represent an active range between two indices.

### Intuition

A brute-force approach usually checks many unnecessary combinations.

```java icon="code" title="Brute Force Pair Scan"
// Brute Force: O(n^2)
for (int i = 0; i < nums.length; i++) {
    for (int j = i + 1; j < nums.length; j++) {
        if (nums[i] + nums[j] == target) {
            return new int[] { i, j };
        }
    }
}
```

Two Pointers reduces this by moving intelligently instead of testing every pair. In many problems, we can move from **O(n^2)** to **O(n)** after sorting (or when sorted order is already given).

### What are Predictable Dynamics?

This pattern works best when pointer movement has a predictable effect.

* In a **sorted array**, moving `left` rightward increases (or keeps) value.
* Moving `right` leftward decreases (or keeps) value.
* In read/write scenarios, moving `fast` scans candidates while `slow` stores valid output.

Because pointer movement changes state in a known way, each decision can safely discard many impossible candidates.

### Core Strategies

#### 1. Inward Traversal (Opposite Ends)

* **Setup:** `left = 0`, `right = n - 1`.
* **Movement:** Move inward based on current comparison.
* **Use Cases:** Two Sum II, Valid Palindrome, Container With Most Water.

#### 2. Unidirectional Traversal (Fast & Slow)

* **Setup:** both pointers near start.
* **Movement:** `fast` explores; `slow` marks the next valid position.
* **Use Cases:** Remove duplicates, move zeros, partitioning tasks.

#### 3. Staged Traversal

* **Setup:** one pointer drives discovery, second pointer activates later.
* **Movement:** second pointer runs only after a trigger is found.
* **Use Cases:** Expand-around-center palindrome checks, segment processing.

### Implementation Examples

#### Example A: Prefix-Based Rolling Computation

```java icon="code" title="Linear Data Structures - Prefix Running Sum"
public int[] runningSum(int[] nums) {
    int[] prefix = new int[nums.length];
    int running = 0;

    for (int i = 0; i < nums.length; i++) {
        running += nums[i];
        prefix[i] = running;
    }

    return prefix;
}
```

#### Example B: Sliding-Style Window Scan

```java icon="code" title="Linear Data Structures - Window Template"
public int maxSumWindow(int[] nums, int k) {
    int left = 0;
    int windowSum = 0;
    int best = Integer.MIN_VALUE;

    for (int right = 0; right < nums.length; right++) {
        windowSum += nums[right];

        if (right >= k - 1) {
            best = Math.max(best, windowSum);
            windowSum -= nums[left];
            left++;
        }
    }

    return best;
}
```


### Real-World Analogies

#### 1. Conceptual Analogy: Quality Review Team

One reviewer scans rapidly for valid records (`fast`), while another maintains the approved list (`slow`). The scanner never rewrites data; the approver decides what stays.

#### 2. Technical Analogy: Memory Compaction in Runtime Systems

A runtime scanner walks heap memory and identifies live objects. A second pointer tracks the next free compacted slot. Live objects are moved forward, leaving garbage behind.

### When to Use (and How to Recognize It)

* Input is linear: array, string, linked list.
* Problem asks for pairs, symmetry, or in-place cleanup.
* Data is sorted (or can be sorted first).
* You see constraints pushing away from nested loops.

### Pros & Cons

| Pros | Cons |
| --- | --- |
| Often reduces **O(n^2)** scans to **O(n)** traversal. | Many variants require sorted input or preprocessing. |
| Usually needs **O(1)** extra space. | Pointer movement rules can be easy to get wrong at boundaries. |
| Clean for in-place transformations. | Not naturally suited for non-linear structures. |
