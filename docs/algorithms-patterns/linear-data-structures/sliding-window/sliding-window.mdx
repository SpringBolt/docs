### Introduction

The **Sliding Window** pattern processes contiguous portions of an array or string by maintaining a moving window `[left, right]`.

Instead of recomputing every subarray/substring from scratch, we incrementally update the window as it moves.

### Intuition

Brute force checks all contiguous ranges, which is expensive.

```java icon="code" title="Brute Force Subarray Scan"
// Brute Force: O(n^2)
for (int start = 0; start < nums.length; start++) {
    int sum = 0;
    for (int end = start; end < nums.length; end++) {
        sum += nums[end];
        // Evaluate subarray nums[start..end]
    }
}
```

Sliding Window avoids repeated work:

* Add the new right element.
* Remove the old left element (when needed).
* Keep rolling state (sum, frequency map, distinct count, etc.).

### What are Predictable Dynamics?

The key dynamic is **contiguity**. When window boundaries move by one step:

* Only one element enters from the right.
* Only one element leaves from the left.
* Window state can be updated in O(1) or near O(1).

This predictable “add/remove” behavior enables linear-time scans.

### Core Strategies

#### 1. Fixed-Size Window

* Window size `k` never changes.
* Once size hits `k`, slide both pointers.
* Use Cases: max sum of size `k`, average of size `k`.

#### 2. Variable-Size Window (Expand/Shrink)

* Expand right pointer to satisfy condition.
* Shrink left pointer while condition remains valid.
* Use Cases: smallest subarray with sum ≥ target.

#### 3. Constraint-Tracking Window

* Maintain counts/maps for characters or categories.
* Shrink while violating constraints (duplicates, too many distinct chars).
* Use Cases: longest substring without repeating chars.

### Implementation Examples

#### Example A: Prefix-Based Rolling Computation

```java icon="code" title="Linear Data Structures - Prefix Running Sum"
public int[] runningSum(int[] nums) {
    int[] prefix = new int[nums.length];
    int running = 0;

    for (int i = 0; i < nums.length; i++) {
        running += nums[i];
        prefix[i] = running;
    }

    return prefix;
}
```

#### Example B: Sliding-Style Window Scan

```java icon="code" title="Linear Data Structures - Window Template"
public int maxSumWindow(int[] nums, int k) {
    int left = 0;
    int windowSum = 0;
    int best = Integer.MIN_VALUE;

    for (int right = 0; right < nums.length; right++) {
        windowSum += nums[right];

        if (right >= k - 1) {
            best = Math.max(best, windowSum);
            windowSum -= nums[left];
            left++;
        }
    }

    return best;
}
```


### Real-World Analogies

#### 1. Conceptual Analogy: Moving Flashlight

Imagine scanning a wall with a flashlight. The illuminated region is your window. As you move right, new area enters the light while old area leaves.

#### 2. Technical Analogy: TCP Receive Buffer Monitoring

A network stack tracks a rolling region of incoming packets and acknowledgements. As packets are processed, the left boundary advances; new packets extend the right boundary.

### When to Use (and How to Recognize It)

* Keywords: **contiguous**, **subarray**, **substring**.
* Need max/min/longest/shortest over contiguous ranges.
* Brute force suggests nested loops over intervals.
* A rolling metric can be maintained incrementally.

### Pros & Cons

| Pros | Cons |
| --- | --- |
| Converts many contiguous-range problems from **O(n^2)** to **O(n)**. | Less useful when data is non-contiguous (subsequence problems). |
| Reuses previous computation efficiently. | Variable-window invariants can be tricky to debug. |
| Works well for streams and online processing. | Some variants need hash maps with non-trivial edge handling. |

### Summary Table

| Strategy | Mechanics | Common Problems |
| --- | --- | --- |
