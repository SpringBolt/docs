### Introduction

The **LCS Pattern** compares two sequences and builds a DP table over two indices.

State `dp[i][j]` typically means answer for prefixes `s1[0..i-1]` and `s2[0..j-1]`.

### Intuition

If last characters match, extend subsequence by 1.
If not, best comes from dropping one character from either string.

### Implementation Examples

#### Example A: LCS Length

```java icon="code" title="DP LCS - Length"
public int longestCommonSubsequence(String text1, String text2) {
    int n = text1.length();
    int m = text2.length();
    int[][] dp = new int[n + 1][m + 1];

    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= m; j++) {
            if (text1.charAt(i - 1) == text2.charAt(j - 1)) {
                dp[i][j] = 1 + dp[i - 1][j - 1];
            } else {
                dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
            }
        }
    }

    return dp[n][m];
}
```

#### Example B: Delete Operation for Two Strings

```java icon="code" title="DP LCS - Min Deletions to Equal"
public int minDistance(String word1, String word2) {
    int lcs = longestCommonSubsequence(word1, word2);
    return (word1.length() - lcs) + (word2.length() - lcs);
}
```

---

### When to Use

* Two-string comparison problems.
* "Subsequence" keyword (not substring).
* Need alignment-style DP.

### Pros & Cons

| Pros | Cons |
| --- | --- |
| Very general two-sequence template. | 2D DP memory can be large. |
| Clear recurrence and base initialization. | Reconstruction needs extra bookkeeping. |

### Summary Table

| Variant | Time | Space |
| --- | --- | --- |
| LCS | $O(nm)$ | $O(nm)$ |
| LCS (Space Optimized) | $O(nm)$ | $O(min(n,m))$ |
| Edit Distance | $O(nm)$ | $O(nm)$ |
