### Introduction

The **Fibonacci Pattern** is the foundational Dynamic Programming pattern where state `dp[i]` depends on a fixed number of previous states.

Typical recurrence: `dp[i] = dp[i - 1] + dp[i - 2]`.

It teaches memoization, tabulation, and space optimization.

### Intuition

If a problem repeats the same subproblems and each answer depends on previous indices, store results once and reuse them.

This converts exponential recursion to linear DP.

---

### Core Variations

#### 1. Top-Down (Memoization)
Recursion + cache.

#### 2. Bottom-Up (Tabulation)
Build from base cases to target.

#### 3. Space-Optimized DP
Keep only last few states.

---

### Implementation Examples

#### Example A: Climbing Stairs (Tabulation)

```java icon="code" title="DP Fibonacci Pattern - Climbing Stairs"
public int climbStairs(int n) {
    if (n <= 2) return n;

    int[] dp = new int[n + 1];
    dp[1] = 1;
    dp[2] = 2;

    for (int i = 3; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }

    return dp[n];
}
```

#### Example B: Space Optimized

```java icon="code" title="DP Fibonacci Pattern - Space Optimized"
public int climbStairsOptimized(int n) {
    if (n <= 2) return n;

    int prev2 = 1;
    int prev1 = 2;

    for (int i = 3; i <= n; i++) {
        int current = prev1 + prev2;
        prev2 = prev1;
        prev1 = current;
    }

    return prev1;
}
```

---

### When to Use

* State depends on fixed previous positions.
* Recurrence like `f(n-1)` and `f(n-2)`.
* Need counting/ways problems.

### Pros & Cons

| Pros | Cons |
| --- | --- |
| Easy gateway to DP thinking. | Can be overused where greedy works. |
| Simple transitions and base cases. | Incorrect base cases break whole recurrence. |

### Summary Table

| Approach | Time | Space |
| --- | --- | --- |
| Memoization | $O(n)$ | $O(n)$ |
| Tabulation | $O(n)$ | $O(n)$ |
| Space Optimized | $O(n)$ | $O(1)$ |
