### Introduction

The **Linear Search** pattern scans elements one by one until the target is found or input ends.

It does not require sorted data and works on arrays, lists, strings, and streams.

This makes it a default baseline when no ordering or index structure is available.

### Intuition: "Check in Sequence"

Given array `[8, 4, 1, 9, 3]`, target `9`:

1. Check `8` -> not target.
2. Check `4` -> not target.
3. Check `1` -> not target.
4. Check `9` -> found.

No jumping or elimination happens. We simply inspect in order.

---

### Core Variations

#### 1. Exact Match in Unsorted Array

* Return index when `arr[i] == target`.
* Return `-1` if not found.

#### 2. First Occurrence by Predicate

* Search for first element satisfying a condition.
* Useful when target is not a single value.

#### 3. Last Occurrence Scan

* Traverse full array and keep latest matching index.
* Useful for log/event data.

### Implementation Examples

#### Example A: Classic Linear Search

**Problem:** Return index of target in integer array.

```java icon="code" title="Linear Search - Exact Match"
public int linearSearch(int[] nums, int target) {
    for (int i = 0; i < nums.length; i++) {
        if (nums[i] == target) {
            return i;
        }
    }
    return -1;
}
```

#### Example B: First Element Greater Than Target

**Problem:** Return first index where value is greater than target.

```java icon="code" title="Linear Search - First Matching Predicate"
public int firstGreaterThan(int[] nums, int target) {
    for (int i = 0; i < nums.length; i++) {
        if (nums[i] > target) {
            return i;
        }
    }
    return -1;
}
```

#### Example C: Last Occurrence

**Problem:** Return last index of target in array.

```java icon="code" title="Linear Search - Last Occurrence"
public int lastOccurrence(int[] nums, int target) {
    int answer = -1;

    for (int i = 0; i < nums.length; i++) {
        if (nums[i] == target) {
            answer = i;
        }
    }

    return answer;
}
```

#### Example D: 2D Matrix Linear Search

**Problem:** Return true if target exists in matrix (no sorted guarantee).

```java icon="code" title="Linear Search - 2D Matrix"
public boolean searchMatrixLinear(int[][] matrix, int target) {
    for (int r = 0; r < matrix.length; r++) {
        for (int c = 0; c < matrix[r].length; c++) {
            if (matrix[r][c] == target) {
                return true;
            }
        }
    }

    return false;
}
```

### Real-World Analogy

Think of checking attendance on a printed sheet.

* Start from the top.
* Read names line by line.
* Stop when the name is found.

That is linear search behavior.

---

### When to Use

Look for these clues:

* Input is **unsorted**.
* Dataset is small or single-pass stream.
* Need simple and reliable baseline.
* Cost of sorting is not justified.

### Complexity Analysis

* Best Case: **$O(1)$** (target at first position).
* Average / Worst Case: **$O(n)$**.
* Space: **$O(1)$**.

### Pros & Cons

| Pros | Cons |
| --- | --- |
| **No Preconditions:** Works without sorting. | **Slower on Large Data:** Worst case scans all elements. |
| **Simple:** Easy to implement and debug. | **No Logarithmic Speedup:** Cannot eliminate search space. |
| **Flexible:** Can search by custom condition/predicate. | **Repeated Queries Costly:** Better to preprocess/index if many queries. |

### Summary Table

| Variant | Time | Space | Typical Use |
| --- | --- | --- | --- |
| **Exact Match (1D)** | $O(n)$ | $O(1)$ | Unsorted array lookup |
| **First Predicate Match** | $O(n)$ | $O(1)$ | Constraint-based first hit |
| **Last Occurrence** | $O(n)$ | $O(1)$ | Last matching position |
| **2D Linear Search** | $O(m \cdot n)$ | $O(1)$ | Matrix without sorted guarantees |
