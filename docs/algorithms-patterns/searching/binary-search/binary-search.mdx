### Introduction

The **Binary Search** pattern is used to find a target value in a **sorted** or **monotonic** search space by repeatedly cutting the space in half.

Instead of scanning linearly, we compare with the middle element and eliminate half of the remaining candidates at each step.

This gives logarithmic performance: **$O(\log n)$**.

### Intuition: "Eliminate Half Every Time"

Given sorted array `[1, 3, 5, 7, 9, 11]`, target `7`:

1. Check middle (`5`) -> target is bigger.
2. Ignore left half.
3. Check middle of right half (`9`) -> target is smaller.
4. Ignore right part.
5. Land on `7`.

Each comparison discards about 50% of the remaining search range.

---

### Core Variations

#### 1. Exact Match in 1D Sorted Array

* Find index where `arr[mid] == target`.
* Return `-1` if not found.

#### 2. First / Last Occurrence (Duplicates)

* Continue searching left or right after a match.
* Useful for range boundaries.

#### 3. Lower Bound / Upper Bound

* Lower Bound: first index `>= target`.
* Upper Bound: first index `> target`.
* Useful for insertion points and frequency counting.

#### 4. Search in 2D Sorted Matrix

* Treat matrix as virtual sorted 1D array.
* Convert `mid` to `(row, col)`.

#### 5. Search in Rotated Sorted Array

* At least one half is always sorted.
* Detect sorted half and decide which side to keep.

### Implementation Examples

#### Example A: Classic Binary Search (1D Exact Match)

**Problem:** Find target index in a sorted integer array.

```java icon="code" title="Binary Search - 1D Exact Match"
public int search(int[] nums, int target) {
    int left = 0;
    int right = nums.length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] == target) {
            return mid;
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return -1;
}
```

#### Example B: First Occurrence in Sorted Array

**Problem:** Return first index of target in sorted array with duplicates.

```java icon="code" title="Binary Search - First Occurrence"
public int firstOccurrence(int[] nums, int target) {
    int left = 0;
    int right = nums.length - 1;
    int answer = -1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] == target) {
            answer = mid;
            right = mid - 1; // keep searching left
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return answer;
}
```

#### Example D: Lower Bound (First Index >= Target)

**Problem:** Return first index where value is greater than or equal to target.

```java icon="code" title="Binary Search - Lower Bound"
public int lowerBound(int[] nums, int target) {
    int left = 0;
    int right = nums.length; // half-open range [left, right)

    while (left < right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid;
        }
    }

    return left;
}
```

#### Example E: Upper Bound (First Index > Target)

**Problem:** Return first index where value is strictly greater than target.

```java icon="code" title="Binary Search - Upper Bound"
public int upperBound(int[] nums, int target) {
    int left = 0;
    int right = nums.length; // half-open range [left, right)

    while (left < right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] <= target) {
            left = mid + 1;
        } else {
            right = mid;
        }
    }

    return left;
}
```

#### Example F: Search in 2D Sorted Matrix

**Problem:** Given an `m x n` matrix where each row is sorted and first element of each row is greater than last element of previous row, return true if target exists.

```java icon="code" title="Binary Search - 2D Matrix"
public boolean searchMatrix(int[][] matrix, int target) {
    int rows = matrix.length;
    int cols = matrix[0].length;

    int left = 0;
    int right = rows * cols - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;
        int r = mid / cols;
        int c = mid % cols;
        int value = matrix[r][c];

        if (value == target) {
            return true;
        } else if (value < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return false;
}
```

#### Example G: Search in Rotated Sorted Array

**Problem:** Find target index in sorted array rotated at unknown pivot.

```java icon="code" title="Binary Search - Rotated Sorted Array"
public int searchRotated(int[] nums, int target) {
    int left = 0;
    int right = nums.length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] == target) {
            return mid;
        }

        // Left half is sorted
        if (nums[left] <= nums[mid]) {
            if (nums[left] <= target && target < nums[mid]) {
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
        // Right half is sorted
        else {
            if (nums[mid] < target && target <= nums[right]) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
    }

    return -1;
}
```

### Real-World Analogy

Think of looking up a word in a physical dictionary.

* You open near the middle.
* Based on alphabetical order, you ignore half of the pages.
* Repeat until exact word is found.

That is Binary Search in daily life.

---

### When to Use

Look for these clues:

* Input is **sorted** or monotonic.
* Need fast lookup on large search space.
* Terms like **first true**, **minimum feasible**, **lower bound**, **insertion index**.

### Complexity Analysis

* Exact Match / First / Last / Lower / Upper Bound: **Time $O(\log n)$, Space $O(1)$**.
* 2D Matrix Binary Search: **Time $O(\log (m \cdot n))$, Space $O(1)$**.
* Rotated Array Search: **Time $O(\log n)$, Space $O(1)$**.

### Pros & Cons

| Pros | Cons |
| --- | --- |
| **Very Fast:** $O(\log n)$ lookup. | **Requires Order:** Needs sorted/monotonic structure. |
| **Scalable:** Works well for huge arrays/ranges. | **Boundary Bugs:** Off-by-one errors are common. |
| **Flexible:** Many variants share same core template. | **Edge Cases:** Duplicates, pivot logic, and bounds need care. |

### Summary Table

| Variant | Time | Space | Typical Use |
| --- | --- | --- | --- |
| **1D Exact Match** | $O(\log n)$ | $O(1)$ | Search value in sorted array |
| **First / Last Occurrence** | $O(\log n)$ | $O(1)$ | Duplicates and range queries |
| **Lower / Upper Bound** | $O(\log n)$ | $O(1)$ | Insert position, counting frequency |
| **2D Matrix Search** | $O(\log (m \cdot n))$ | $O(1)$ | Row-wise globally sorted matrix |
| **Rotated Array Search** | $O(\log n)$ | $O(1)$ | Pivoted sorted arrays |
