### Introduction

The **Merge Intervals** pattern is used when problems involve overlapping ranges like `[start, end]`.

The general idea:
1. Sort intervals by start time.
2. Traverse once.
3. Merge when overlap exists.

This pattern appears in scheduling, calendar conflicts, meeting rooms, and timeline optimization.

### Intuition: "Combine What Touches"

Suppose we have intervals:
`[[1,3], [2,6], [8,10], [15,18]]`

After sorting (already sorted here), compare each interval with the last merged one:

* `[1,3]` and `[2,6]` overlap (`2 <= 3`) → merge to `[1,6]`
* `[8,10]` does not overlap with `[1,6]` → start new interval
* `[15,18]` does not overlap with `[8,10]` → start new interval

Final output: `[[1,6], [8,10], [15,18]]`

Instead of checking every pair, we only compare with the last merged interval.

---

### Core Variations

#### 1. Standard Merge

* Merge all overlapping intervals.
* Return condensed non-overlapping list.

#### 2. Insert Interval

* Insert one interval into sorted list.
* Merge where needed and keep final order.

#### 3. Intersection of Two Interval Lists

* Two sorted lists.
* Use two pointers to compute overlap regions.

### Implementation Examples

#### Example A: Merge Overlapping Intervals

**Problem:** Given an array of intervals where `intervals[i] = [start, end]`, merge all overlapping intervals.

```java icon="code" title="Merge Intervals - Merge Overlaps"
public int[][] merge(int[][] intervals) {
    if (intervals == null || intervals.length <= 1) {
        return intervals;
    }

    Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

    List<int[]> merged = new ArrayList<>();
    int[] current = intervals[0];
    merged.add(current);

    for (int i = 1; i < intervals.length; i++) {
        int[] next = intervals[i];

        // Overlap exists if next start <= current end
        if (next[0] <= current[1]) {
            current[1] = Math.max(current[1], next[1]);
        } else {
            current = next;
            merged.add(current);
        }
    }

    return merged.toArray(new int[merged.size()][]);
}
```

#### Example B: Insert Interval

**Problem:** Given a sorted non-overlapping interval list, insert a new interval and merge if required.

```java icon="code" title="Merge Intervals - Insert Interval"
public int[][] insert(int[][] intervals, int[] newInterval) {
    List<int[]> result = new ArrayList<>();
    int i = 0;

    // 1. Add all intervals that end before newInterval starts
    while (i < intervals.length && intervals[i][1] < newInterval[0]) {
        result.add(intervals[i]);
        i++;
    }

    // 2. Merge all overlapping intervals with newInterval
    while (i < intervals.length && intervals[i][0] <= newInterval[1]) {
        newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
        newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
        i++;
    }
    result.add(newInterval);

    // 3. Add remaining intervals
    while (i < intervals.length) {
        result.add(intervals[i]);
        i++;
    }

    return result.toArray(new int[result.size()][]);
}
```

### Real-World Analogy

Think of booking a **meeting room calendar**:

* If one meeting is `10:00-11:00` and next is `10:30-12:00`, they overlap and block one continuous slot `10:00-12:00`.
* Non-overlapping meetings stay separate.

The merged schedule gives the true occupied blocks.

---

### When to Use

Look for these clues:

* Input is a list of **ranges/intervals**.
* Words like **overlap, merge, conflict, schedule, timeline**.
* Need to minimize or consolidate ranges.

### Pros & Cons

| Pros | Cons |
| --- | --- |
| **Efficient:** Sorting + one pass solves many interval tasks. | **Sorting Cost:** Usually requires $O(n \log n)$. |
| **Simple Logic:** Compare only with last merged interval. | **Mutation Risk:** In-place edits can cause side effects if not careful. |
| **Reusable Template:** Same skeleton applies to insert/intersection variants. | **Order Dependency:** Must sort (unless guaranteed sorted). |

### Summary Table

| Variant | Key Idea | Time | Space |
| --- | --- | --- | --- |
| **Merge Intervals** | Sort by start, merge overlaps | $O(n \log n)$ | $O(n)$ |
| **Insert Interval** | Add left, merge middle, add right | $O(n)$ (sorted input) | $O(n)$ |
| **Interval Intersection** | Two-pointer scan on two sorted lists | $O(n + m)$ | $O(1)$ excluding output |
