### Introduction

**Heap Sort** uses a binary heap to sort elements in-place with guaranteed **$O(n \log n)$** time.

It first builds a max-heap, then repeatedly extracts the maximum element to the end of the array.

### Intuition: "Repeatedly Remove the Maximum"

1. Build max-heap so largest element is at root.
2. Swap root with last unsorted element.
3. Reduce heap size and heapify root.
4. Repeat until heap size is 1.

---

### Implementation Example

```java icon="code" title="Heap Sort"
public void heapSort(int[] nums) {
    int n = nums.length;

    for (int i = n / 2 - 1; i >= 0; i--) {
        heapify(nums, n, i);
    }

    for (int end = n - 1; end > 0; end--) {
        int temp = nums[0];
        nums[0] = nums[end];
        nums[end] = temp;

        heapify(nums, end, 0);
    }
}

private void heapify(int[] nums, int heapSize, int root) {
    int largest = root;
    int left = 2 * root + 1;
    int right = 2 * root + 2;

    if (left < heapSize && nums[left] > nums[largest]) {
        largest = left;
    }

    if (right < heapSize && nums[right] > nums[largest]) {
        largest = right;
    }

    if (largest != root) {
        int temp = nums[root];
        nums[root] = nums[largest];
        nums[largest] = temp;
        heapify(nums, heapSize, largest);
    }
}
```

### Real-World Analogy

Like maintaining a leaderboard where the top score is always at the front. Remove top, re-balance, and repeat.

---

### When to Use

* Need guaranteed **$O(n \log n)$** worst-case.
* Need in-place sorting with constant extra array memory.
* Stability is not required.

### Why Choose Heap Sort Over Quick Sort Sometimes

* No quadratic worst-case time.
* More predictable runtime under adversarial inputs.

### Pros & Cons

| Pros | Cons |
| --- | --- |
| Guaranteed $O(n \log n)$ in all cases. | Not stable. |
| In-place with $O(1)$ extra space (excluding recursion stack). | Usually slower constants than Quick Sort in practice. |
| Good worst-case reliability. | Poor cache locality compared to Quick Sort. |

### Complexity Analysis

| Case | Time | Space |
| --- | --- | --- |
| Best | $O(n \log n)$ | $O(1)$ |
| Average | $O(n \log n)$ | $O(1)$ |
| Worst | $O(n \log n)$ | $O(1)$ |
