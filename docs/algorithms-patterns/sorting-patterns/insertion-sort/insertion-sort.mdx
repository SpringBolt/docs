### Introduction

**Insertion Sort** builds the sorted portion of an array one element at a time.

For each new element, it shifts larger elements right and inserts the current value in the correct position.

It is highly effective for small or nearly sorted datasets.

### Intuition: "Insert Current Card into Sorted Hand"

Starting from index `1`:

* treat left side as sorted,
* pick current value,
* shift larger values right,
* place current value into its sorted gap.

---

### Implementation Example

```java icon="code" title="Insertion Sort"
public void insertionSort(int[] nums) {
    for (int i = 1; i < nums.length; i++) {
        int current = nums[i];
        int j = i - 1;

        while (j >= 0 && nums[j] > current) {
            nums[j + 1] = nums[j];
            j--;
        }

        nums[j + 1] = current;
    }
}
```

---

### Real-World Analogy

Like sorting cards in your hand:

* keep left side sorted,
* insert each new card into the right position by shifting cards.

---

### When to Use

* Input is already or almost sorted.
* Small arrays where low constant factors matter.
* As a base-case optimization in advanced sorts (like Quick Sort, TimSort).

### Why It Works Well for Nearly Sorted Data

* Number of shifts equals number of inversions.
* Few inversions means close to linear behavior.

### Pros & Cons

| Pros | Cons |
| --- | --- |
| In-place and stable. | Quadratic worst case. |
| Excellent on nearly sorted data. | Not suitable for large random arrays. |
| Very low overhead. | Many shifts when data is reverse sorted. |

### Complexity Analysis

| Case | Time | Space |
| --- | --- | --- |
| Best (already sorted) | $O(n)$ | $O(1)$ |
| Average | $O(n^2)$ | $O(1)$ |
| Worst (reverse sorted) | $O(n^2)$ | $O(1)$ |
