### Introduction

**Sorting Patterns** are foundational techniques for arranging data in increasing or decreasing order.

Different sorting algorithms optimize for different constraints:

* small vs large input size,
* nearly sorted vs random input,
* memory usage limits,
* value range constraints,
* stability requirements.

There is no single best sorting algorithm for every case.

---

### Core Sorting Families

#### 1. Comparison-Based Sorting

* Compare two elements to decide order.
* Lower bound is **$O(n \log n)$** for general sorting.
* Examples: Merge Sort, Quick Sort, Heap Sort.

#### 2. Exchange / Incremental Sorting

* Build order via swaps or shifts.
* Very useful for small or nearly sorted arrays.
* Examples: Bubble Sort, Selection Sort, Insertion Sort, Shell Sort.

#### 3. Non-Comparison Sorting

* Use value properties (digits/range/buckets) instead of pairwise comparisons.
* Can beat **$O(n \log n)$** under right constraints.
* Examples: Counting Sort, Radix Sort, Bucket Sort.

---

### Choosing the Right Sorting Pattern

| Situation | Good Choice | Why |
| --- | --- | --- |
| Data is almost sorted | **Insertion Sort** | Fast in practice with low movement. |
| Need guaranteed $O(n \log n)$ | **Merge Sort / Heap Sort** | Predictable worst-case performance. |
| Average-case speed on arrays | **Quick Sort** | Cache-friendly and typically very fast. |
| Integers in small known range | **Counting Sort** | Linear-time counting approach. |
| Fixed-length numeric keys | **Radix Sort** | Sorts by digits with stable passes. |
| Values uniformly distributed | **Bucket Sort** | Near-linear expected performance. |
| Tiny arrays / educational | **Bubble / Selection** | Simple to implement and reason about. |
