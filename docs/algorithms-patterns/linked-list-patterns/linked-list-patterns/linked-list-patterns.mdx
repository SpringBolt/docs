### Introduction

The **Linked List Patterns** family groups reusable pointer techniques for singly and doubly linked list questions.

Most linked-list problems are not about arithmetic on values. They are about **safe link rewiring** with minimal extra memory.

This section is split into separate detailed files so each standard pattern can be studied independently.

### Shared Safety Rules

Before modifying links:

1. Save references (`nextNode`, `groupNext`, etc.) before rewiring.
2. Change one pointer at a time.
3. Keep a stable anchor (`dummy`, `prev`, `tail`) for reconnecting list segments.
4. Validate boundary conditions (`head == null`, `k <= 1`, short list lengths).

These rules prevent broken chains, lost nodes, and accidental cycles.
